2. Azure AI Content Safety ⭐ único da Azure

O Bedrock oferece Guardrails (filtros acoplados ao modelo), mas não tem um serviço dedicado de classificação de conteúdo com severidade graduada por categoria, usável fora de qualquer LLM. A Azure tem. Cada texto recebe severidade 0–6 em 4 categorias:

Capacidade sem equivalente direto no Bedrock. Roda com make safety sobre 10 textos de exemplo (infra + casos sintéticos).

3. Semantic Kernel — framework de agentes nativo da Azure

Equivalente ao Bedrock Agents. Aparece com frequência em vagas de IA corporativa Microsoft. O modelo decide automaticamente quais funções chamar (function calling = o "planner" moderno): 

InfraPlugin.compare_services("ECS", "AKS")
RunbookPlugin.search_runbook("rollback")   # → chama o retriever do azure_rag (RAG)

Pergunta: "Como faço rollback de ECS e diferença para AKS?"
→ SK planner → search_runbook + compare_services → resposta sintetizada + fontes

7. Quando escolher cada nuvem

Escolha Azure / AI Foundry

  • Stack Microsoft (Entra ID, Office, .NET)
  • Compliance + moderação dedicada (Content Safety)
  • Agentes corporativos com Semantic Kernel
  • Melhor custo/qualidade em chat (GPT-4o-mini)

Continue na AWS / Bedrock

  • Claude é o modelo principal (topo em raciocínio/código)
  • Já há peso de workloads/identidade/rede na AWS
  • Quer RAG gerenciado de ponta a ponta (Knowledge Bases)

Multicloud é viável: a camada de prompts/LLM é portável; o lock-in real está nos dados e no RAG gerenciado.
